# R1: 비대면·자동화 수익 아이디어 12개 발굴

작성일: 2026-05-29
작업 ID: t_dbf53141
대상 자산: 주말아이 `https://jumalikids.com/`, repo `/home/hermes/projects/weekend-kids`

## 결론 먼저

상위 5개 추천은 아래 순서입니다.

- 1순위: 주말아이 핵심 롱테일 행사 허브
  - 이유: 이미 구축된 데이터 파이프라인과 가장 맞고, Search Console/AdSense 준비도 개선과 바로 연결됩니다.
- 2순위: 날씨·미세먼지 기반 실내 대안 페이지
  - 이유: 부모의 검색 의도가 강하고, 기존 행사 데이터에 기상·대기 데이터를 얹는 방식이라 자동화 난이도가 낮습니다.
- 3순위: 예약·요금·연령 체크 자동 요약
  - 이유: “무료/예약/연령” 오판 리스크를 줄이면서 사이트 품질을 올립니다. AdSense 승인에도 유리한 신뢰 신호입니다.
- 4순위: 계절 공공시설 운영정보 허브
  - 이유: 어린이날·물놀이장·눈썰매장처럼 시즌 검색 수요가 뚜렷합니다. 페이지 수를 많이 늘리지 않아도 실험이 가능합니다.
- 5순위: 외출 준비물·상황별 체크리스트 PDF
  - 이유: 검색 유입이 적어도 빠르게 디지털 상품 실험을 할 수 있습니다. 다만 판매는 신뢰와 트래픽이 쌓인 뒤가 안전합니다.

다음 후보는 TourAPI 기반 근교 체험 코스입니다. 시장성은 좋지만 사진·동선·장소 정보 검증 부담이 있어 2차 후보로 두는 편이 안전합니다.

## 전제와 계산 방식

- 사람과 직접 계약하는 대행·컨설팅 모델은 제외했습니다.
- 무단 크롤링, 저작권 침해, 얇은 AI 글 대량 생산 모델은 제외했습니다.
- 점수는 모두 0~5점입니다.
  - 시장성: 검색 의도와 반복 수요가 강할수록 높음
  - 자동화: API·정형 데이터·정적 생성에 잘 맞을수록 높음
  - 난이도: 5점이 쉬움, 0점이 어려움
  - 수익화: AdSense·제휴·디지털 상품 전환 가능성이 높을수록 높음
  - 리스크: 5점이 낮은 리스크, 0점이 높은 리스크
- AdSense 추정은 Google AdSense의 Page RPM 계산식에 맞춰 `월 페이지뷰 ÷ 1,000 × 가정 RPM`으로 계산했습니다.
- 한국어 정보성 롱테일 사이트의 초기 RPM은 보수적으로 1,200~2,500원 범위로만 가정했습니다. 보장 수익이 아닙니다.
- 제휴 추정은 `페이지뷰 × 클릭률 × 구매전환율 × 객단가 × 수수료율`로 계산했습니다. 실제 수수료율은 제휴 계정에서 매번 확인해야 합니다.
- 디지털 상품 추정은 `랜딩 방문자 × 구매전환율 × 가격`으로 계산했습니다. 초기에는 결제보다 다운로드·이메일 수집 지표를 먼저 봐야 합니다.

## 소스 신호

- 현재 주말아이 계획: 서울 문화행사 정보, 공공서비스예약, 구별·상황별 정적 페이지, AdSense, Search Console 중심 구조.
- Google AdSense 도움말: Page RPM은 예상 수입을 페이지뷰로 나눈 뒤 1,000을 곱하는 방식입니다.
- 서울 열린데이터광장: 서울시 문화행사 정보는 분류, 자치구, 행사명, 장소, 날짜, 기관명, 이용대상, 이용요금, 홈페이지, 문의, 위치, 시작일, 종료일, 유·무료, 행사시간 등을 제공합니다.
- 공공데이터포털/한국관광공사 TourAPI: 전국 관광정보와 사진정보 중 활용 가능한 정보를 OpenAPI로 제공합니다.
- 공공데이터포털/기상청: 단기예보 조회서비스는 초단기실황, 초단기예보, 단기예보 등을 제공합니다.
- 공공데이터포털/에어코리아: 측정소·시도별 대기오염 정보와 대기질 예보 정보를 제공합니다.
- 공공데이터포털/행정안전부: 전국 어린이놀이시설 위치 및 기본 정보 서비스가 있습니다.
- 도서관 데이터: 전국도서관표준데이터는 있으나, 어린이 프로그램 단위 데이터는 지역별로 흩어져 있어 자동화 리스크가 더 큽니다.

## 아이디어 12개

- 1. 주말아이 핵심 롱테일 행사 허브
  - 대상 검색의도: “서울 이번주말 아이랑 무료 행사”, “마포구 어린이 무료 체험”, “강남구 주말 어린이 행사”.
  - 수익 방식: AdSense를 1차로 두고, 나중에 준비물 체크리스트·제휴 링크를 보조로 붙입니다.
  - 자동화 가능성: 높음. 기존 `culturalEventInfo` 수집, 필터링, 구별·상황별 페이지 생성, sitemap 갱신 구조를 그대로 씁니다.
  - 데이터/콘텐츠 소스: 서울 열린데이터광장 문화행사 정보, 서울 공공서비스예약, 자체 작성한 가이드 문장, 공식 행사 링크.
  - 7일 검증 방법: 핵심 롱테일 25개 페이지의 제목·설명·내부링크를 정리하고, Search Console에서 노출·색인 상태를 확인합니다. 동시에 이벤트 50건을 표본 점검해 무료·연령·예약 태그 오판률을 기록합니다.
  - 리스크: AdSense 승인은 느립니다. 행사 데이터의 유료/무료 필드와 요금 문구가 충돌하면 신뢰가 떨어집니다.
  - 점수: 시장성 5 / 자동화 5 / 난이도 4 / 수익화 3 / 리스크 4 = 21
  - 보수적 수익 추정: 월 10,000~50,000 PV, RPM 1,500~2,500원 가정 시 월 15,000~125,000원입니다.

- 2. 날씨·미세먼지 기반 실내 대안 페이지
  - 대상 검색의도: “비오는날 서울 아이랑 실내”, “미세먼지 나쁨 아이랑 갈만한 곳”, “오늘 아이랑 실내 무료 행사”.
  - 수익 방식: AdSense가 기본입니다. 우산, 우비, 물병, 실내놀이 키트 같은 준비물 제휴는 AdSense 승인 이후 제한적으로 붙입니다.
  - 자동화 가능성: 높음. 기상청 단기예보와 에어코리아 대기질 데이터를 행사 태그와 결합해 “오늘/이번 주말 실내 추천” 페이지를 만들 수 있습니다.
  - 데이터/콘텐츠 소스: 기상청 단기예보, 에어코리아 대기오염정보, 서울 문화행사 데이터, 자체 안내 문장.
  - 7일 검증 방법: “비오는날”, “미세먼지”, “실내” 랜딩 10개를 만들고, 실제 비/미세먼지 나쁜 날에 페이지 제목·메타 설명이 검색 의도와 맞는지 수동 확인합니다.
  - 리스크: 날씨와 대기질은 빠르게 바뀝니다. 건강·안전 보장처럼 쓰면 안 되고, “방문 전 공식 안내 확인” 문구가 필요합니다.
  - 점수: 시장성 5 / 자동화 4 / 난이도 4 / 수익화 3 / 리스크 4 = 20
  - 보수적 수익 추정: 월 8,000~40,000 PV, RPM 1,500~2,500원 가정 시 AdSense 월 12,000~100,000원입니다. 제휴는 클릭률 0.5~1.2%, 구매전환율 1~2.5%, 객단가 30,000원, 수수료 3% 가정 시 월 360~10,800원 수준으로만 봅니다.

- 3. 도서관 어린이 프로그램 캘린더
  - 대상 검색의도: “구립도서관 어린이 프로그램”, “초등 독서 프로그램 무료”, “주말 도서관 체험”.
  - 수익 방식: AdSense가 기본입니다. 독서 기록표 PDF 같은 무료 다운로드 후 저가 디지털 상품을 붙일 수 있습니다.
  - 자동화 가능성: 중간 이하입니다. 전국도서관표준데이터는 있지만 프로그램 데이터가 지역별 홈페이지에 흩어져 있어 무단 크롤링을 피해야 합니다.
  - 데이터/콘텐츠 소스: 전국도서관표준데이터, 각 지자체·도서관이 공식 API/RSS/공개 캘린더를 제공하는 경우만 제한 활용, 자체 작성 안내문.
  - 7일 검증 방법: 서울 1개 구만 골라 공식 API나 재사용 가능한 데이터가 있는지 확인합니다. 없으면 “도서관 위치+공식 프로그램 링크 모음” 수준으로 축소합니다.
  - 리스크: 자동화 가능한 프로그램 소스가 부족할 수 있습니다. 수동 등록이 필요해지면 이번 조건과 어긋납니다.
  - 점수: 시장성 4 / 자동화 3 / 난이도 3 / 수익화 2 / 리스크 3 = 15
  - 보수적 수익 추정: 월 5,000~25,000 PV, RPM 1,200~2,000원 가정 시 월 6,000~50,000원입니다.

- 4. TourAPI 기반 근교 무료·저가 체험 코스
  - 대상 검색의도: “서울 근교 아이랑 당일치기 무료”, “아이랑 박물관 무료”, “주말 가족 체험 코스”.
  - 수익 방식: AdSense가 기본입니다. 돗자리, 간식통, 보냉백, 유모차 용품 등 제휴는 보조입니다.
  - 자동화 가능성: 중상입니다. TourAPI 관광정보와 주말아이 행사 데이터를 결합해 “장소 1곳+근처 무료 행사+날씨 주의” 형태의 코스 페이지를 만들 수 있습니다.
  - 데이터/콘텐츠 소스: 한국관광공사 TourAPI, 서울·경기 공공행사 데이터, 기상청 예보, 공식 장소 링크.
  - 7일 검증 방법: 서울 근교 10개 코스만 만들고, 사진 저작권·공식 링크·위치·휴무일 표시를 점검합니다. 자동 코스가 억지스럽다면 장소형 페이지로 축소합니다.
  - 리스크: 사진 라이선스와 휴무·주차 정보가 민감합니다. 길찾기 정확도를 보장하면 안 됩니다.
  - 점수: 시장성 4 / 자동화 4 / 난이도 3 / 수익화 3 / 리스크 4 = 18
  - 보수적 수익 추정: 월 8,000~40,000 PV, RPM 1,500~2,500원 가정 시 AdSense 월 12,000~100,000원입니다. 제휴는 클릭률 0.5~1.0%, 구매전환율 1~2%, 객단가 50,000원, 수수료 3% 가정 시 월 600~12,000원입니다.

- 5. 외출 준비물·상황별 체크리스트 PDF
  - 대상 검색의도: “아이랑 나들이 준비물”, “아기랑 외출 준비물”, “비오는날 아이랑 준비물”.
  - 수익 방식: 무료 체크리스트로 이메일이나 재방문을 만들고, 3,900~5,900원 PDF 묶음이나 인쇄용 템플릿을 판매합니다. 제휴 링크는 보조입니다.
  - 자동화 가능성: 높음. 날씨, 연령, 실내/실외, 행사 시간에 따라 체크리스트 문구를 조합할 수 있습니다.
  - 데이터/콘텐츠 소스: 자체 제작 체크리스트 템플릿, 주말아이 행사 태그, 날씨 데이터. 외부 글 복붙 없이 자체 문장으로 구성합니다.
  - 7일 검증 방법: 무료 PDF 1개와 유료 예고 랜딩 1개를 만들고, 다운로드율·이메일 입력률·구매 버튼 클릭률을 봅니다. 결제는 실제 상품 준비 후에만 켭니다.

  - 점수: 시장성 3 / 자동화 4 / 난이도 5 / 수익화 3 / 리스크 4 = 19

- 6. 예약·요금·연령 체크 자동 요약
  - 대상 검색의도: “어린이 행사 예약 필요”, “무료 어린이 체험 예약”, “몇 살부터 가능한 행사”.
  - 수익 방식: 직접 판매보다 AdSense와 사이트 체류시간 개선 효과가 큽니다. 핵심 행사 페이지의 신뢰도를 높이는 보강 모델입니다.

  - 리스크: 가장 큰 리스크는 잘못된 무료·연령 안내입니다. 애매하면 “공식 페이지 확인 필요”로 낮춰야 합니다.
  - 점수: 시장성 4 / 자동화 4 / 난이도 4 / 수익화 3 / 리스크 4 = 19
  - 보수적 수익 추정: 월 6,000~30,000 PV, RPM 1,500~2,500원 가정 시 월 9,000~75,000원입니다. 단독 수익보다 기존 페이지 전환·체류 개선 효과가 목적입니다.

  - 대상 검색의도: “근처 어린이놀이터”, “유아 놀이터 위치”, “아이랑 갈만한 놀이터”.
  - 수익 방식: AdSense가 기본입니다. 외출 준비물 제휴는 낮은 비중으로만 검토합니다.
  - 자동화 가능성: 중상입니다. 전국 어린이놀이시설 기본 정보는 공공데이터로 다룰 수 있지만, 실제 상태·그늘·화장실 같은 편의 정보는 자동화가 어렵습니다.
  - 데이터/콘텐츠 소스: 행정안전부 전국어린이놀이시설정보서비스, 지자체 공원 데이터, 공식 위치 정보.
  - 7일 검증 방법: 서울 3개 구만 대상으로 위치 페이지를 만들고, 데이터 필드가 검색 의도에 충분한지 확인합니다. 편의정보가 없으면 과감히 후순위로 둡니다.
  - 리스크: “안전”을 보장하면 위험합니다. 데이터 최신성과 현장 상태 차이가 큽니다.
  - 점수: 시장성 4 / 자동화 4 / 난이도 3 / 수익화 2 / 리스크 3 = 16
  - 보수적 수익 추정: 월 5,000~25,000 PV, RPM 1,200~2,000원 가정 시 월 6,000~50,000원입니다.

- 8. 방학 무료 체험·캠프 시즌 허브
  - 대상 검색의도: “여름방학 무료 체험”, “겨울방학 초등 프로그램”, “초등 방학 캠프 무료”.
  - 수익 방식: 시즌 AdSense가 기본입니다. 방학 계획표 PDF, 준비물 체크리스트 같은 디지털 상품을 붙일 수 있습니다.
  - 자동화 가능성: 중간입니다. 행사·예약·도서관·공공기관 데이터가 섞여 있고 시즌마다 소스가 바뀝니다.
  - 데이터/콘텐츠 소스: 서울 문화행사 정보, 공공서비스예약, 도서관·박물관 공식 공개 데이터, 자체 작성 시즌 가이드.
  - 7일 검증 방법: 다음 방학 시즌용 랜딩 5개를 먼저 만들고, “무료/저가/예약마감/연령” 필터가 실제 데이터에서 작동하는지 확인합니다.
  - 리스크: 계절성이 강하고 마감 정보가 빨리 바뀝니다. 오래된 페이지는 자동으로 숨기거나 업데이트 표시가 필요합니다.
  - 점수: 시장성 5 / 자동화 3 / 난이도 3 / 수익화 3 / 리스크 3 = 17
  - 보수적 수익 추정: 시즌 월 10,000~50,000 PV, RPM 1,500~2,500원 가정 시 AdSense 월 15,000~125,000원입니다. 방학 계획표 PDF는 월 방문자 5,000~30,000명, 구매전환율 0.1~0.25%, 가격 3,900~5,900원 가정 시 월 19,500~442,500원입니다.

- 9. 아이랑 당일치기 루트·소요시간 플래너
  - 대상 검색의도: “지하철로 가는 어린이 박물관”, “아이랑 당일치기 코스”, “차 없이 아이랑 갈만한 곳”.
  - 수익 방식: AdSense와 여행 준비물 제휴를 함께 검토할 수 있습니다.
  - 자동화 가능성: 중간입니다. 장소 데이터, 이동 시간, 날씨, 휴무일을 결합해야 해서 구현 난이도가 높습니다.
  - 데이터/콘텐츠 소스: TourAPI, OpenStreetMap/OSRM 같은 경로 엔진, 기상청 예보, 공식 장소 링크.
  - 7일 검증 방법: 자동 루트 생성은 미루고, “역세권 어린이 장소 10개”처럼 좁혀서 검색 의도와 수동 검증 부담을 봅니다.
  - 리스크: 이동시간·휴무일·주차 정보 오류가 사용자 불만으로 이어질 수 있습니다. 지도 API 약관도 확인해야 합니다.
  - 점수: 시장성 4 / 자동화 3 / 난이도 2 / 수익화 4 / 리스크 3 = 16
  - 보수적 수익 추정: 월 8,000~40,000 PV, RPM 1,500~2,500원 가정 시 AdSense 월 12,000~100,000원입니다. 제휴는 클릭률 0.5~1.0%, 구매전환율 1~2%, 객단가 50,000원, 수수료 3% 가정 시 월 600~12,000원입니다.

- 10. 주간 자동 리포트·뉴스레터
  - 대상 검색의도: 검색보다 재방문 의도입니다. “이번 주말 뭐 하지?”를 매주 해결하는 모델입니다.
  - 수익 방식: 무료 이메일/텔레그램 리포트로 재방문을 만들고, 나중에 광고 없는 유료 요약, PDF 묶음, 제휴 영역을 붙입니다. 사람과 계약하는 스폰서십은 우선 제외합니다.
  - 자동화 가능성: 높음. 매주 금요일에 행사 데이터, 날씨, 미세먼지, 예약 필요 여부를 요약해 발송할 수 있습니다.
  - 데이터/콘텐츠 소스: 주말아이 내부 행사 데이터, 기상청, 에어코리아, Search Console 인기 페이지.
  - 7일 검증 방법: 사이트에 “이번 주말 알림 받기” 입력 폼을 붙이고, 7일 동안 가입 20명 또는 클릭률 2% 이상을 목표로 봅니다.
  - 리스크: 개인정보 수집·수신동의·수신거부 처리가 필요합니다. 초기 트래픽이 적으면 가입 수가 거의 없을 수 있습니다.
  - 점수: 시장성 3 / 자동화 4 / 난이도 3 / 수익화 3 / 리스크 4 = 17
  - 보수적 수익 추정: 무료 구독자 300~1,000명, 유료 전환 0.5~1.5%, 월 3,900원 가정 시 월 5,850~58,500원입니다. 초기에는 매출보다 구독 전환율이 핵심입니다.

- 11. 계절 공공시설 운영정보 허브
  - 대상 검색의도: “한강 물놀이장 개장”, “서울 눈썰매장 아이랑”, “어린이날 무료행사”, “가을 축제 아이랑”.
  - 수익 방식: 시즌성 AdSense가 기본입니다. 준비물 체크리스트와 제휴를 보조로 붙일 수 있습니다.
  - 자동화 가능성: 중상입니다. 반복 시즌 페이지 구조는 자동화할 수 있지만, 운영기간·요금·휴장일은 공식 출처 확인이 필요합니다.
  - 데이터/콘텐츠 소스: 서울 문화행사 정보, 지자체 공식 공지, 공공시설 공개 데이터, 공식 운영 페이지 링크. 무단 크롤링 대신 공식 API·공개 파일·수동 seed를 씁니다.
  - 7일 검증 방법: 한 계절 주제 하나만 고릅니다. 예를 들어 여름 물놀이장 10개 페이지를 만들고, 공식 링크·운영기간·요금 표시 정확도를 확인합니다.
  - 리스크: 시즌이 지나면 트래픽이 급감합니다. 공식 정보 변경에 민감하므로 “마지막 확인일”과 자동 만료 처리가 필요합니다.
  - 점수: 시장성 4 / 자동화 4 / 난이도 4 / 수익화 3 / 리스크 4 = 19
  - 보수적 수익 추정: 시즌 월 10,000~50,000 PV, RPM 1,500~2,500원 가정 시 월 15,000~125,000원입니다.

- 12. 공공데이터 미니사이트 스타터킷
  - 대상 검색의도: “공공데이터 사이트 만들기”, “정적 사이트 자동 배포”, “Cloudflare Pages 공공데이터 템플릿”.
  - 수익 방식: 주말아이 구축 경험을 일반화한 디지털 템플릿을 9,900~19,000원에 판매합니다. 고객별 제작 대행은 제외합니다.
  - 자동화 가능성: 높음. 기존 Python 수집·정적 생성·sitemap·Cloudflare Pages 구조를 샘플 데이터와 문서로 패키징할 수 있습니다.
  - 데이터/콘텐츠 소스: 자체 코드, 자체 문서, 공개 샘플 데이터. 주말아이의 비밀키·운영 데이터는 제외합니다.
  - 7일 검증 방법: 무료 샘플 repo 또는 설명 랜딩을 만들고, “다운로드/메일 등록/구매 의향 클릭”만 확인합니다. 실제 판매 전에는 설치 문서를 충분히 줄여야 합니다.
  - 리스크: 대상자가 부모가 아니라 제작자라 시장이 작습니다. 설치 지원 요구가 생기면 비대면·자동화 조건과 충돌할 수 있어 “지원 없음”을 명확히 해야 합니다.
  - 점수: 시장성 2 / 자동화 4 / 난이도 3 / 수익화 3 / 리스크 4 = 16
  - 보수적 수익 추정: 월 랜딩 방문자 300~1,500명, 구매전환율 0.3~0.8%, 가격 9,900~19,000원 가정 시 월 8,910~228,000원입니다.

## 실행 순서 제안

- 1주차: 1번, 2번, 6번을 묶어서 진행합니다.
  - 이유: 모두 기존 행사 데이터 품질과 페이지 신뢰도를 높이는 작업입니다.
  - 성공 기준: 신규·개선 페이지 30개 이상, 오판 표본 점검 50건, Search Console 노출 시작 또는 색인 요청 가능 상태.
- 2주차: 11번 계절 허브를 한 주제만 실험합니다.
  - 이유: 시즌성 검색어는 작은 페이지 묶음으로도 반응을 볼 수 있습니다.
  - 성공 기준: 공식 링크 10개 이상, 만료일/마지막 확인일 표시, sitemap 반영.
- 3주차: 5번 체크리스트 PDF를 무료 다운로드부터 붙입니다.
  - 이유: 초기 트래픽이 작아도 다운로드율을 보면 디지털 상품 가능성을 빠르게 판단할 수 있습니다.
  - 성공 기준: 다운로드 20건 또는 랜딩 클릭률 1% 이상.

## 보류 또는 후순위

- 도서관 프로그램 캘린더는 시장성은 있지만 자동화 소스가 불안정합니다. 공식 API/RSS가 확인되는 구부터만 진행해야 합니다.
- 놀이터 지도는 검색 수요가 있지만 “안전” 표현 리스크가 큽니다. 안전 보장이 아니라 위치·공식 기본정보 제공으로 제한해야 합니다.
- 당일치기 루트 플래너는 수익화 여지는 있으나 길찾기·휴무·라이선스 검증 부담이 큽니다. 장소형 페이지로 먼저 축소 검증하는 편이 좋습니다.
- 공공데이터 스타터킷은 완전 비대면 디지털 상품이지만, 대상 시장이 부모가 아니라 제작자입니다. 주말아이 본체가 안정된 뒤 부산물로 검토하는 편이 안전합니다.
