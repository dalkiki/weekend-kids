# 주말아이 MVP 구축 계획서

> **목표:** “전국/지역별 아이랑 갈만한 무료 행사·체험·도서관 프로그램”을 자동 수집·정리해 구글 검색 유입과 애드센스 수익을 만드는 저비용 자동화 사이트를 구축한다.
>
> **핵심 조건:** ① 구축 후 손이 적게 감 ② 사람과 계약하지 않음 ③ 애드센스 같은 자동 정산형 수익 ④ 레드오션 블로그가 아니라 공공데이터 기반 롱테일 정보 사이트

---

## 0. 확정 사항

- **사이트명:** 주말아이
- **도메인 방향:** 유료 도메인 구매 전, 무료 주소로 MVP를 먼저 테스트한다.
- **사이트 톤:** 깔끔한 정보형. 단, 부모가 읽기 쉽게 너무 딱딱하지 않은 문장으로 작성한다.
- **MVP 지역:** 서울
- **1차 데이터 소스:** 서울 열린데이터광장 “서울시 문화행사 정보”

---

## 1. 한 줄 컨셉

**주말아이**는 부모가 “이번 주말 아이랑 어디 가지?”라고 검색했을 때, 지역·날짜·무료 여부·실내/실외·예약 여부를 바로 볼 수 있게 해주는 자동 갱신형 정보 사이트다.

예시 검색어:

- 서울 이번주말 아이랑 무료 행사
- 마포구 아이랑 무료 체험 이번주
- 비오는날 서울 아이랑 실내 무료 행사
- 경기 어린이 도서관 무료 프로그램
- 강남구 주말 어린이 체험 예약

---

## 2. 왜 이 아이템인가

### 조건 적합성

- **자동화:** 공공데이터/API → 정리 → 정적 페이지 생성 → 배포까지 자동화 가능
- **계약 없음:** 사장님 영업/고객 대응 없이 AdSense 중심 수익화 가능
- **블루오션성:** “지역 + 이번주말 + 아이 + 무료 + 실내/예약” 롱테일 조합은 수작업 블로그가 다 커버하기 어려움
- **저비용:** Cloudflare Pages/GitHub Pages 등 무료 호스팅 가능. 초기 유료비는 도메인 정도
- **확장성:** 서울 MVP → 경기 → 전국 → 도서관/박물관/공공예약 데이터 확장 가능

### 피해야 할 방향

- AI가 대충 쓴 일반 블로그 글 100개 만들기
- 복붙·얇은 자동 페이지 대량 생성
- 출처 없는 행사 정보 재게시
- 의료/안전처럼 틀리면 위험한 정보부터 시작하기

---

## 3. MVP 범위

### 1차 MVP는 “서울”만 한다

처음부터 전국을 하면 데이터 품질 관리가 어려우므로, 1차는 서울로 제한한다.

1차 목표:

- 서울 문화행사/공공예약/도서관 어린이 프로그램 중심
- 무료 또는 저비용 행사 우선
- 이번 주/이번 주말/다음 주말 페이지 자동 생성
- 구별 페이지 자동 생성
- AdSense 승인 가능한 기본 콘텐츠와 사이트 구조 확보

### 1차 핵심 페이지

- `/` 홈
- `/seoul/this-weekend/` 서울 이번 주말 아이랑 무료 행사
- `/seoul/indoor/` 서울 비오는날 아이랑 실내 무료 행사
- `/seoul/free/` 서울 아이랑 무료 행사 모음
- `/seoul/gu/{구이름}/` 강남구/마포구/송파구 등 구별 행사
- `/events/{event-id}/` 개별 행사 상세 페이지
- `/about/` 소개
- `/contact/` 문의
- `/privacy/` 개인정보처리방침
- `/sources/` 데이터 출처 안내

## 4. 데이터 소스 후보

### 우선순위 A: 서울시 문화행사 정보

- 출처: 서울 열린데이터광장 “서울시 문화행사 정보”
- 제공 정보: 분류, 자치구, 행사명, 장소, 날짜, 기관명, 이용대상, 이용요금, 홈페이지, 문의, 위치, 시작일, 종료일, 유/무료, 행사시간 등
- 장점: MVP에 가장 잘 맞음
- 사용 목적: 기본 행사 데이터 수집

### 우선순위 B: 서울시 문화행사 공공서비스예약 정보

- 출처: 서울 열린데이터광장 “서울시 문화행사 공공서비스예약 정보”
- 장점: 예약 필요 여부/바로가기 URL과 연결 가능
- 사용 목적: 예약형 체험 프로그램 보강

### 우선순위 C: 문화공공데이터광장

- 전국 박물관/미술관/공연/행사 데이터 후보
- 사용 목적: 서울 MVP 이후 전국 확장

### 우선순위 D: 도서관 프로그램 데이터

- 서울도서관/구립도서관/공공데이터 후보
- 사용 목적: “도서관 어린이 무료 프로그램” 페이지 강화

## 5. 자동화 구조

### 추천 기술 구조

- 저장소: GitHub repo `weekend-kids` 또는 `jumal-i`
- 사이트 생성: Python + Jinja2 또는 Astro
- 데이터 저장: JSON 파일 또는 SQLite
- 배포: Cloudflare Pages 또는 GitHub Pages
- 자동 실행: GitHub Actions 매일 새벽 4시
- 수익화: Google AdSense
- 검색 등록: Google Search Console

### 자동화 흐름

```text
매일 새벽 4시
→ 서울시/공공데이터 API 호출
→ 원본 데이터 저장
→ 중복 제거 및 필터링
→ 어린이/가족/무료/실내 키워드 태깅
→ 날짜 지난 행사 제외
→ 지역별·상황별 페이지 생성
→ sitemap.xml 갱신
→ Cloudflare Pages 자동 배포
```

### Hermes가 도울 부분

- 공공데이터 API 조사
- 수집 스크립트 작성
- 페이지 템플릿 작성
- SEO 제목/설명 생성
- 애드센스 승인용 기본 페이지 작성
- GitHub repo 정리
- 배포 자동화
- 주간 품질 점검 자동화

## 6. 수익화 전략

### 1단계: AdSense

초기 목표는 애드센스 승인과 검색 유입 확보.

승인 전 필수 요소:

- 원본성 있는 안내문 포함
- 사이트 소개 페이지
- 개인정보처리방침
- 문의 페이지
- 데이터 출처 페이지
- 모바일 친화적 레이아웃
- 얇은 자동 페이지 방지

### 2단계: 제휴 링크는 나중에 선택

사람과 계약하지 않는 범위에서 가능:

- 쿠팡파트너스: 돗자리, 물병, 유아 간식, 우비, 유모차 용품 등
- 단, 초반에는 광고성 과다 노출을 피하고 AdSense 승인 후 천천히 붙인다.

### 3단계: 디지털 상품은 선택사항

나중에 유입이 생기면:

- 주말 나들이 체크리스트 PDF
- 아이랑 외출 준비물 체크리스트
- 비오는날 실내놀이 리스트

하지만 1차 MVP에는 넣지 않는다.

## 8. 태훈님이 직접 해야 하는 일 — 아주 쉬운 버전

### A. 도메인 정하기

시프티가 후보를 뽑아주면, 태훈님은 마음에 드는 이름만 고르면 된다.

추천 후보:

- `jumali.kr`
- `weekendkids.kr`
- `jumalchild.com`
- `kidsweekend.kr`
- `todaykids.kr`

직접 할 일:

1. 후보 중 마음에 드는 이름 고르기
2. 도메인을 살지 말지 결정하기
3. 결제는 태훈님이 직접 하기

쉬운 설명:

> 도메인은 사이트 주소다. 예를 들어 `jumali.kr` 같은 것. 보통 1년에 1~2만 원 정도다. 처음에는 무료 주소로 테스트하고, 애드센스 신청 전 유료 도메인을 붙이는 방식도 가능하다.

---

### B. Google AdSense 계정 준비

직접 할 일:

1. https://www.google.com/adsense/ 접속
2. Google 계정으로 로그인
3. 사이트가 준비된 뒤 사이트 주소 입력
4. 심사 신청
5. 승인/거절 결과를 시프티에게 알려주기

주의:

- 사이트가 비어 있을 때 신청하면 거절될 가능성이 높다.
- 최소한 기본 페이지와 충분한 유용한 콘텐츠가 생긴 뒤 신청한다.
- 거절되어도 흔하다. 거절 사유를 보고 보완하면 된다.

---

### C. Google Search Console 등록

직접 할 일:

1. https://search.google.com/search-console 접속
2. 사이트 주소 등록
3. 소유권 확인
4. sitemap.xml 제출

쉬운 설명:

> Search Console은 “구글아, 내 사이트 여기 있어. 검색에 가져가줘.”라고 알려주는 도구다. 사이트맵은 사이트의 페이지 목록이다.

시프티가 도와줄 수 있는 부분:

- 소유권 확인 파일 생성
- sitemap 주소 알려주기
- 제출해야 할 주소 안내

---

### D. API 키 발급

공공데이터/서울 열린데이터광장 API 키가 필요할 수 있다.

직접 할 일:

1. 서울 열린데이터광장 가입
2. 인증키 발급
3. 발급된 키를 시프티에게 전달

주의:

- API 키는 비밀번호처럼 취급한다.
- 텔레그램에 보내면 시프티가 `.env`에 저장하고, 답변에는 다시 노출하지 않는다.

---

### E. 최종 확인만 하기

태훈님이 매일 할 필요는 없다. 대신 초기에 이런 판단만 해주면 된다.

- 사이트 이름 마음에 드는지
- 글 톤이 너무 딱딱하지 않은지
- 부모 입장에서 보기 쉬운지
- 광고가 너무 거슬리지 않는지
- 무료/유료 구분이 잘 되는지

## 11. 초기 품질 기준

MVP라도 아래 기준은 지켜야 한다.

### 각 자동 페이지에 필요한 것

- 페이지 제목이 검색어와 잘 맞을 것
- 최종 업데이트 날짜 표시
- 데이터 출처 표시
- 행사명/날짜/장소/요금/대상/예약 여부 표시
- “방문 전 공식 페이지 확인” 안내
- 빈 페이지는 생성하지 않기
- 행사 수가 너무 적으면 인접 카테고리와 병합

### 상세 페이지에 필요한 것

- 행사명
- 날짜/시간
- 장소
- 자치구
- 비용
- 대상 연령 또는 이용대상
- 예약 필요 여부
- 공식 링크
- 지도 링크 후보
- 유의사항

## 14. 태훈님용 초간단 요약

태훈님이 직접 매일 글 쓰는 프로젝트가 아니다.

시프티가 만들 구조는:

```text
공공데이터에서 행사 가져오기
→ 아이랑 갈만한 무료/실내/주말 행사만 정리
→ 지역별 페이지 자동 생성
→ 구글 검색 유입 받기
→ 애드센스 수익화
```

태훈님이 직접 할 일은 대부분 계정/결제/승인 같은 것뿐이다.

```text
1. 사이트 이름 고르기
2. 도메인 살지 결정하기
3. 공공데이터 API 키 발급하기
4. 구글 서치콘솔 등록하기
5. 애드센스 신청하기
6. 가끔 사이트 보기
```

나머지 개발·자동화·문서화는 시프티가 맡는다.
